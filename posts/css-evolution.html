Web design evolution with CSS

When the web started, layouts looked like this article. A single page with text and images following the only top-down flow available with the only comfort of background and text colors.

Table era

Then tables came along. It was a huge step forward. Now, you were able to organize your page in a two dimensional way. Tables unleashed creativity on the web.

One of the reasons is that design conventions on the web did not exist, so everything had to be invented. Things that we call convention are just practices that stays over time. For instance, no one is able to tell who and when navbar and sidebar became navigation layout conventions.

Another reason for this creativity was the absence of one big constraint. I'm talking about screen sizes. Back at that time they were no smartphone, no tablet and almost all computers shared the same screen size. Without this constraint you were sure your visual didn't break.

However, one of the major problems was that the visual styling and the content were twisted together and that made your page impossible to redesign and also impossible to reuse your style. In this context, CSS came out.

Hard coded float layouts era

CSS was invented to separate the content from the visual style, given us a lot of advantages among reusability, accessibility and SEO.

CSS did not provide a powerful layout model nevertheless, float did the job for a while. At the same time, the complexity caused by the arrival of new devices and screens, was growing. With it, the need of a reliable responsive design became a necessity.

The problem was, Float wasn't designed to be used like we did and we ended up playing with percentage to create multi column layouts. Lots of math had to be done in order to align elements, in addition to this the support of older browser was a nightmare. Even with the best intentions our layouts lacked of reliability. So people started to lose confidence in CSS and concluded CSS wasn't enough.

CSS frameworks era

That's why, CSS frameworks came along, and with them the idea of grids. The 12 column grid layout was one of the most famous ones. This idea is to split your screen in 960px width container split in 12 columns. Frameworks took away the burden of responsiveness and did the math for you. You can now organize your elements and to be sure your design won't break. Websites heavily used them to get rid of layout headaches. But it came with a price. Standardization has removed creativity not to mention that we have to learn another tool in order to create layout for the web.

Statement

Standardization has removed creativity. A large majority of the web follows the same layout patterns, which has been even enforced by the misusage of design systems. Of corse, best practices and industrialization are very important. But lots us use these tools without understanding their philosophy and what tradeoffs they imply.

Take a look at newspaper layouts. Every pages has it's own layout wish depend on what content it displays. Newspapers use layouts to emphases the content and to make it attractive. They wants their readers to enjoy the reading experience through the content and the visual. The complete opposite of this boring article.
